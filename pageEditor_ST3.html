Feedback Page

The feedback page was designed to provide users with a clear and engaging way to explore gender equality opportunities and submit feedback. The page contains three main parts: a scrollable programme directory, a ratings and reviews section, and an interactive feedback form. I used a consistent pastel blue, pink, and green visual theme so that the page matched the rest of the website while still feeling approachable and eye catchy.

HTML Structure

The page uses a clear semantic structure with <header>, <nav>, <main>, multiple <section> elements, and a <footer>. Inside the programme directory, I used <article> cards for each opportunity so that the content was grouped logically. For the form, I used <form>, <fieldset>, and <legend> to separate personal details from feedback details, which helped make the structure clearer and easier to use.

CSS Techniques

I used CSS Grid to organise the programme cards, review cards, and form rows. Rounded corners, shadows, gradients, and consistent spacing were applied to create a soft and readable design. Hover effects were used on programme buttons, navigation links, and submit/reset buttons to give immediate visual feedback. I also created a scrollable area for the programme cards so that users could browse the opportunities without the section becoming too long.

Design Choices

I chose a card based layout because it allowed the programmes and reviews to be presented in a structured and visually balanced way. The scrollable programme section was useful because it improved usability by keeping the page compact. The feedback form was designed with clear labels and section grouping so users could understand it easily. JavaScript was used to add a live character counter, check user input, highlight errors and display a confirmation message for submission.

Team Page

The team page was created to introduce the people behind the website and explain each member’s role and contribution. The main purpose of this page was to present team information in a more interactive way rather than using a simple list of names.

HTML Structure

The page uses the same shared structure with a header, navigation bar, main content area, and footer. Within the main section, I used a team grid containing one block for each group member. Each block includes an image, the member’s name, role, pages/features contributed to, and a short contribution summary. The details are placed inside an overlay panel so that extra information can appear within the same card.

CSS Techniques

CSS grid was used to make the four team member cards look evenly displayed. I used transitions, transform, and opacity to create a hover and focus effect. When the user hovers over or focuses on a card, the image becomes faded and slightly enlarged, while the overlay panel slides up into view. This created a smoother and more interactive presentation while keeping the design visually consistent with the rest of the website.

Design Choices

I chose an overlay layout because it allowed the team details to stay hidden until needed, which made the page cleaner and more visually engaging. The hover and focus states were important because the coursework specifically asked for interactive thumbnails and expandable details. I also added tabindex="0" to each interactive team block so keyboard users can access the same content, which improved accessibility.

Content Page

My content page focuses on 'Representations of Women in Media' explaining how media can shape views, reinforce stereotypes, and influence how women are viewed in society. The page is organised into clear sections with relevant images and headings, making the information easy to understand while also keeping it engaging and accessible for users.

HTML Structure

I structured the page using one main <h1> title and several <h2> section headings. Each major content area was placed inside its own <section> element with a unique id. At the top of the page, I included an internal navigation area using anchor links so users could jump directly to the section they wanted. I also added a “Go to Top” button linked to the top of the page using an internal anchor.

CSS Techniques

I kept the same pastel theme as my other pages so the content page looked consistent with the site. Page-specific CSS was used for the internal navigation box, image cards, highlight box, mini-cards, and the fixed “Go to Top” button. I used grid layout in the two-column section and responsive image styling with borders, rounded corners, and shadows to make the images support the content visually.

Design Choices

I used internal navigation because the page contains several sections and the brief required meaningful internal linking for longer pages. The “Go to Top” button was included to improve usability and make it easier to move back to the top without scrolling manually. I chose a mix of text sections, lists, and images so the content would feel more engaging and not appear too dense. The content was written in an original way and focused on action, fairness, and student relevance.

Challenges and Lessons Learned

One challenge I faced was keeping all three pages visually consistent while still making each one look appropriate for its own purpose. I solved this by reusing the same colour palette, rounded sections, and navigation style across my pages, while adding page specific elements such as overlays, scroll containers, and internal links where needed.

Another challenge was making sure the feedback form had enough JavaScript interactivity to meet the coursework requirements. I fixed this by adding a live character counter, custom error messages, invalid field highlighting, and a confirmation message after successful validation. This helped me understand how JavaScript can be used to improve the user experience rather than relying only on HTML form validation.

I also learned the importance of checking file names, navigation links, and footer links carefully. Small mistakes in link names can stop pages opening properly, so testing each page repeatedly was necessary. Overall, this coursework improved my understanding of semantic HTML, CSS layout techniques, accessibility, and how to combine structure, styling, and interactivity in a more organised way.

Accessibility Considerations

Accessibility was considered throughout my pages in several ways. I used semantic HTML structure so the content is organised clearly using headers, navigation, sections, and footers. This makes the pages easier to read and more understandable.

All images include descriptive alt text so users who cannot see the images can still understand their purpose. Form fields on the feedback page have clear labels, and related inputs are grouped using fieldset and legend to improve usability. On the team page, keyboard accessibility was improved by adding tabindex="0" to each interactive team card and using focus styles alongside hover effects.

I also used readable font sizing, clear colour contrast, visible section separation, and structured headings. These choices help users scan the pages more easily and make the pages more inclusive.
