Splash Screen

For the splash screen, I created a simple full-screen layout using HTML that includes a title, a short introduction, and some extra information. I used CSS to position everything in the centre of the screen and added a background image with a slight overlay so the text is still easy to read. I also used CSS transitions to create a fade-out effect so the page doesn’t just suddenly change. JavaScript was used to add a countdown timer and automatically redirect the user to the home page after a few seconds. I also added a skip button so users have the option to go straight to the main site if they don’t want to wait.

Action Impact Simulator

For the Action Impact Simulator, I built the layout using HTML with a section for the introduction, a set of action cards, and an area to display the results. Each card includes an image, a title, and a short description, along with a value stored using a data attribute. I used CSS to style the cards with shadows, hover effects, and spacing so they look more interactive, and I designed the results section to stand out by centring it and adding a border. JavaScript was used to make the simulator work by allowing users to click on cards to select or deselect them, which updates the total score. It also changes the impact level and message depending on the score, making the page more interactive and engaging.

Content Page

s For my content page on Women in STEM, I focused on keeping the layout clear and easy to follow. I used HTML to organise the page into sections, each with its own heading and content, and made sure to use proper heading order so everything is structured properly. I also added images to support the information and included alt text for accessibility. For the styling, I used CSS to space everything out, centre the content, and add background colours and rounded edges so each section stands out. I didn’t use much JavaScript on this page since it’s mainly informational, but I did include internal links so users can quickly jump to different parts of the page.

Accessibility Considerations

Accessibility was considered throughout the development of my pages. I used semantic HTML elements such as main, section, and proper heading levels to ensure the structure is clear for screen readers. All images include meaningful alt text so that users who rely on assistive technologies can understand the content. I also ensured there was sufficient colour contrast between text and background, especially on the splash screen where a gradient overlay was used to improve readability. Additionally, navigation links are clearly labelled to make the website easier to use.

JANET Compliance

To ensure my pages follow web standards, I validated my HTML and CSS using online validation tools. This helped me identify and fix issues such as incorrect heading structure and missing attributes. I also ensured that my code follows good practices, such as using semantic elements and properly structured layouts. By addressing validation errors, I improved both the quality and accessibility of my website.
