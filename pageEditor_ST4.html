Overview

My role in this project was to design and develop the User Profile page, the Sitemap, and my content page on the gender pay gap. I focused on creating pages that are interactive, visually consistent, and meaningful to the user while still fitting into the shared group design. I aimed to make my pages not only functional but also engaging, ensuring that users could easily navigate through the content and understand the purpose of each page. For example, I introduced interactivity in the User Profile page through step-by-step prompts and a progress system, which helps guide users and improves their overall experience. I also ensured consistency across my pages by using a brown accent colour scheme and similar layout structures. This helped create a clear visual identity for my work while still matching the overall theme of the website. Additionally, I focused on making my content meaningful and relevant. In my content page, I explored the gender pay gap in a more personal and engaging way, highlighting not just the facts but also the real-world impact on women. Overall, my goal was to create pages that are clear, interactive, and informative, while also encouraging users to reflect on the importance of gender equality.

User Profile

For the User Profile page, I designed an interactive experience using JavaScript to guide users through a structured, step-by-step process. I divided the profile into three stages (basic details, views and interests, and actions and goals) so that the task feels manageable and less overwhelming for the user. One of my main priorities was improving user experience, so I included a progress bar and a live summary section. This allows users to clearly see how much of their profile has been completed and how their responses are being used, making the interaction feel more meaningful and responsive. I also implemented input validation to improve data quality and usability. The age field only accepts numeric values, while the name and role fields only accept alphabetic characters. This ensures that the data entered is accurate and prevents user errors. From a design perspective, I used a card-based layout with clear spacing and a consistent brown accent colour scheme. I scoped my CSS using page-specific classes to ensure that my styling did not affect other group members’ pages. This demonstrates good practice in collaborative web development.

I also implemented validation so age only accepts numbers, and name/role only accept letters. This improves data accuracy.

Sitemap

For the Sitemap page, I used SVG to create a structured and visually clear representation of the website’s layout. I chose SVG because it allows precise positioning and ensures that the diagram remains scalable and sharp on different screen sizes. I organised the sitemap into three clear levels: entry page, main pages, and content pages. I made sure that the order of the main pages matches the navigation bar, as this improves usability and helps users understand how the website is structured. A key focus for me was alignment and spacing. I carefully adjusted connector lines and node positions so that all elements are evenly spaced and visually balanced. This required multiple refinements, especially to ensure that section labels sit correctly between lines. I also applied a consistent brown colour theme to match my other pages, reinforcing a strong visual identity. Interactive elements, such as hover effects and highlighting the current page, were added to make the sitemap more engaging and user-friendly.

Content Page

My content page focuses on the gender pay gap, which I chose because it is a significant and ongoing issue within gender equality. I wanted my content to go beyond simply presenting facts, so I wrote it in a more personal and reflective tone to highlight how this issue affects real people. I structured the page using clear headings, paragraphs, and lists to guide the reader through the topic. This includes sections explaining what the gender pay gap is, why it exists, its real-world impact, and how it can be reduced. This logical structure improves readability and helps users understand the topic more easily. I also included multiple images throughout the page to break up large sections of text and make the page more visually engaging. This creates a better balance between content and design. One of my key aims was to make the reader think critically about the issue. By combining factual information with a more human perspective, I wanted to emphasise that the gender pay gap is not just a statistic, but something that affects women’s financial independence, opportunities, confidence, and long-term security.

I used headings, lists, and images to structure the page clearly and improve readability.

Accessibility

Accessibility was an important consideration across all of my pages. I used clear headings and a logical structure so that content is easy to follow and navigate. This benefits all users, including those using assistive technologies. I ensured that all images include appropriate alt text so that screen readers can describe the content. I also used readable font sizes, consistent spacing, and good colour contrast to improve visibility and readability. Interactive elements such as buttons and navigation links are clearly styled and easy to identify. On the User Profile page, the step-by-step layout and progress bar help users understand their position within the process, which improves usability and reduces confusion.

JANET Compliance

I ensured that my work complies with JANET regulations by producing appropriate, respectful, and educational content throughout my pages. All information included is relevant to the theme of gender equality and is presented in a responsible manner. I avoided using any offensive or inappropriate material and ensured that my work is original. Where information was influenced by external sources, I made sure it was based on reliable and credible references. Images were used appropriately to support the content and were selected to match the educational purpose of the website. Overall, I ensured that my pages meet the expected standards for ethical and responsible web content.

Challenges

One of the main challenges I faced was working within a shared project while ensuring that my styling did not interfere with other group members’ pages. I solved this by using page-specific classes, such as profile-page and sitemap-page, to scope my CSS. Another challenge was positioning elements accurately within the SVG sitemap. Small changes in coordinates could significantly affect the layout, so I had to carefully adjust spacing, connector lines, and alignment multiple times to achieve a balanced result. I also encountered challenges when implementing JavaScript features, particularly ensuring that user input was handled correctly and displayed dynamically. Through testing and refinement, I was able to improve my understanding of JavaScript logic and debugging. Overall, these challenges helped me develop stronger problem-solving skills and a better understanding of how design and functionality work together in web development.
