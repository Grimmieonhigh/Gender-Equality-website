Shared Template & Navigation

I have created a basic shared template with a colour scheme, a content area that links to everyones content, editor and validation, a global CSS. i have discussed with the team about the colour palette and we have chosen pastel. In the global CSS i have done a basic template so everyone else can input their own CSS changes that they desire (such as header1 or paragraph instead of h2 or p) and added effects to the buttons such as back to top page and the naviagtion page. I have also discussed what parts of the code that should not be changed.

I have made sure that the HTML is easy to understand such as body, main, header, head, footer, back to top. For the CSS techniques, i used card to center and create a section so the team is able to seperate into different parts on their pages. i have used :root so that the colour scheme is easy to access and implement. For paragraph i have used bigger text and letter-spacing so it is easier to read. These choices were used so there is better usability and structure so everyone is able to use the templates.

Challenges i have faced was the navigation page. i decided to put the naviagtion on the side and the logo and title on the left. When i have added more links to the navigation, the links go underneath the logo. It took me a while to figure out that i needed to make the texts much more smaller so it can fit. I have learnt that i have to be careful with the CSS styling because a simple line of code can change how the page is formatted.

Home page

For the home page, I have planned for it to be visually pleasing. I decided to added a banner at the start to introduce what the website is about. I found this particularly hard to implement since the text would not appear on the image or either sepearte itself from the image instead of overlaying or the image would not fit onto the page. to fix this, I changed the padding for the image to fit correctly and restrict the text widths so it is on the left side of the page. I put the image as a background in women-text.

After doing this, I described what the website is about, 3 actions individuals can take and 3 outcomes if these actions are widely adopted. I used it in the card so the text is centered and readable. For the CSS on the header, I added a shadow behind the text so it is more visible.

For the gallery, I wanted to add a glassy effect. I found this a bit difficult due to the colours not blending in correctly. Another problem i have came across was the formatting, the tag for student one and header of the gallery would be out of place and the gallery would not be centered correctly. To fix this, I had to make sure the padding, font-size, letter-spacing are so that the text elements aligned properly and looked balanced. I used grid and centering to make sure the gallery stayed positioned in the middle of the page.

Gallery

For the gallery, I used 4 images to related to the theme of the website such as end violence ect. Each image has a short and simple title and description with a pop up with more details.

For the CSS, i used a grid for the gallery, hover effects to show images are clickable, styling the pop up box and the gallery. This was easy to implement since I did not have to add any extra layout or padding to fit the gallery and images into the page.

For the JavaScript, i used it to detect when the user clicks on the image, show the popup with the correct content, close the popup, change the colours and font when the buttons are clicked. I found Javascript difficult because sometimes the pop up would not work or the colours and fonts would not change. to fix this, I noticed that the current function only sets one colour and does not reset it. It took me a few tries but i was able to find the solution to these problems and help me learn how to use Javascript.

Accessibility Considerations

I have made my pages easy for everyone to use. I added alt text to all images so screen readers can describe them.I used clear headings and simple text so the content is easy to read. I also made sure buttons and links are easy to see and understand. I also used colours that are easy to see such as black text.

JANET Compliance

I followed JANET rules by using appropriate and respectful content. I did not copy work from other websites, and I made sure my work is original. I only used images from free or trusted sources and would include references where needed.
